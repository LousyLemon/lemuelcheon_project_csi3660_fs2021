Apache File Server in CentOS

Project Summary

The purpose of this project was to create a working File Server using Apache and hosting the server within the GCP.
The students would use the concepts learned from the course to create the framework for file transfer between user and server.

Background

This project requires fundamental knowledge of Linux (CentOS) and webpage creation.

  • Basic commands: copy(cp), move(move), make directory(mkdir), create file(touch),
    install, and all the other basics allow for application and utilization of the OS.
  • Linux file system: user permissions, groups, file ownership, config
    files, as these things create the framework for a functioning file server.
  • HTML: is used for webpage creation, design, and implementation of server scripts
    which allow file transfer.

Project Overview

What is the service?

The Apache File Server is a barebones file transfer service, in which users can upload
files to the GCP VM (within the file size limit [~2M]) via their web browser. Offering
easy access and usability. Similar to DropBox, or any other hosted storage system.

Implementing the service

Uploading a file to the server is as simple as clicking on 'Choose File',
selecting a single file you want to upload, then click 'Upload'.
The server will then show the target directory the file has been uploaded
to on the GCP, and whether there were any errors.
